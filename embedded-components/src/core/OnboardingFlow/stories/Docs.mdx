# OnboardingFlow

The **OnboardingFlow** component provides a complete, multi-step onboarding experience for embedded banking applications. It guides users through collecting business information, personal details, and compliance documentation required for KYC/AML verification.

## Features

- **Multi-Step Wizard**: Guided flow with progress tracking
- **Sidebar Navigation**: Optional timeline sidebar for section navigation
- **Client State Management**: Handles new, existing, and returning clients
- **Document Upload**: Integrated document collection with validation
- **NAICS Recommendations**: AI-powered industry code suggestions
- **Theme Support**: Customizable theming for white-label deployments
- **User Event Tracking**: Integration with analytics/RUM systems

## Configuration Variants

### Sidebar Navigation

Enable the sidebar for complex onboarding flows:

```tsx
<OnboardingFlow
  hideSidebar={false}
  availableProducts={['EMBEDDED_PAYMENTS']}
  availableJurisdictions={['US']}
/>
```

### Document Upload Mode

For clients needing to upload additional documents (clientId provided via EBComponentsProvider):

```tsx
<EBComponentsProvider clientId="client-id" apiBaseUrl="/api" headers={{}}>
  <OnboardingFlow docUploadOnlyMode={true} />
</EBComponentsProvider>
```

### Theme Customization

Apply custom theming for white-label deployments:

```tsx
<OnboardingFlow
  theme={{
    colorScheme: 'light',
    variables: {
      primaryColor: '#007bff',
      fontFamily: 'Inter, sans-serif',
    },
  }}
/>
```

## Usage Examples

### Basic Usage

```tsx
import { OnboardingFlow } from '@embedded-components/core/OnboardingFlow';
import { EBComponentsProvider } from '@embedded-components/providers';

// New client onboarding (no clientId)
<EBComponentsProvider apiBaseUrl="/api" headers={{}}>
  <OnboardingFlow
    availableProducts={['EMBEDDED_PAYMENTS']}
    availableJurisdictions={['US']}
  />
</EBComponentsProvider>

// Resume existing client (clientId from provider)
<EBComponentsProvider clientId="0030000133" apiBaseUrl="/api" headers={{}}>
  <OnboardingFlow
    availableProducts={['EMBEDDED_PAYMENTS']}
    availableJurisdictions={['US']}
  />
</EBComponentsProvider>
```

### With Event Handlers

```tsx
<OnboardingFlow
  availableProducts={['EMBEDDED_PAYMENTS']}
  availableJurisdictions={['US']}
  onPostClientSettled={(data, error) => {
    if (data) console.log('Client created:', data.id);
  }}
  onPostPartySettled={(data, error) => {
    if (data) console.log('Party added:', data.id);
  }}
  onGetClientSettled={(data, status, error) => {
    console.log('Client fetch status:', status);
  }}
/>
```

### With User Tracking

```tsx
<OnboardingFlow
  availableProducts={['EMBEDDED_PAYMENTS']}
  availableJurisdictions={['US']}
  userEventsHandler={(context) => {
    analytics.track(context.actionName, {
      ...context.metadata,
      timestamp: context.timestamp,
    });
  }}
/>
```
