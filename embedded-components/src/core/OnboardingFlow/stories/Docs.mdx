# OnboardingFlow

**OnboardingFlow** is the primary React experience for **client onboarding (KYC/KYB)** in this library: incremental business and controller collection, due diligence (operational) questions, document requests and attestations, optional **Account Validation Service (AVS)** bank linking with microdeposit verification when needed, and in-flow review—styled with your **`EBComponentsProvider`** theme and content tokens.

This matches the **Client onboarding (KYC / KYB)** card on the library **Introduction** page (sidebar: **Introduction**), including the **Testing** maturity badge. Scenario labels follow the **SellSense** demo (`client-next-ts`): **New Seller - Onboarding**, **Onboarding - Docs Needed**, **Onboarding - Seller with prefilled data**; active seller flows there compose **Accounts**, **LinkedAccountWidget**, **TransactionsDisplay**, **RecipientsWidget**, **PaymentFlow** (**Make Payment** in the shell), and **ClientDetails** (**Linked Bank Account**, **Seller with Limited DDA**, **Seller with Payments DDA**).

Use **Storybook** (sidebar: **Core → OnboardingFlow**) to preview screens and copy before you integrate.

## Features

- **Incremental onboarding**: Create or resume a client; forms save progress through parties and sections
- **Due diligence & operational questions**: Dynamic questions from your configured API
- **Documents & attestations**: Fulfill requests, upload files (including **document-only mode** when you only need that slice)
- **Optional bank linking**: AVS and microdeposit flows inside the journey or via **LinkedAccountWidget**—same capability as the SellSense **Linked Bank Account** scenario on Introduction (**Testing**)
- **Status-aware UI**: Applicant experience adapts by API status (see **Client States** below and Storybook **Client States**)
- **Industry (NAICS)**: Hierarchical industry classification and suggestions—the same NAICS capability called out under **Utility Components → Industry Classification** on Introduction (**Available**)
- **Monitoring-ready**: Hooks for analytics / RUM consistent with **Integration Scenarios** on Introduction

## Storybook guide

Open **Core → OnboardingFlow** in Storybook and pick the area that matches what you want to see:

1. **Start with Client States** when you need to show stakeholders **what applicants see at each status** (new application, under review, more documents needed, approved, declined).
2. **Use Configuration & variants** to explore **products, regions, sidebar, disclosures, languages, document-only mode, bank-link step, navigation confirmations, and layout height**—the knobs most integrations tune first.
3. **Try Interactive Workflows** for **guided walkthroughs** of filling sections end-to-end (personal, business, owners, operational questions).
4. **Use Entry points** when you want to **land on a specific step** (for example overview, a section, review, or upload documents) without clicking through from the start.

## Integration examples

Common prop combinations you can copy into your app (many have matching examples under **Configuration & variants** in Storybook).

### Sidebar Navigation

Enable the sidebar for complex onboarding flows:

```tsx
<OnboardingFlow
  hideSidebar={false}
  availableProducts={['EMBEDDED_PAYMENTS']}
  availableJurisdictions={['US']}
/>
```

### Document Upload Mode

For clients needing to upload additional documents (clientId provided via EBComponentsProvider):

```tsx
<EBComponentsProvider clientId="client-id" apiBaseUrl="/api" headers={{}}>
  <OnboardingFlow docUploadOnlyMode={true} />
</EBComponentsProvider>
```

### Theme Customization

Theming is applied on **`EBComponentsProvider`** (same pattern as **Introduction → Quick Start → With Custom Theme**):

```tsx
<EBComponentsProvider
  apiBaseUrl="/api"
  headers={{}}
  theme={{
    colorScheme: 'light',
    variables: {
      primaryColor: '#007bff',
      fontFamily: 'Inter, sans-serif',
    },
  }}
>
  <OnboardingFlow
    availableProducts={['EMBEDDED_PAYMENTS']}
    availableJurisdictions={['US']}
  />
</EBComponentsProvider>
```

## Usage Examples

### Basic Usage

```tsx
import {
  EBComponentsProvider,
  OnboardingFlow,
} from '@jpmorgan-payments/embedded-finance-components';

// New client onboarding (no clientId)
<EBComponentsProvider apiBaseUrl="/api" headers={{}}>
  <OnboardingFlow
    availableProducts={['EMBEDDED_PAYMENTS']}
    availableJurisdictions={['US']}
  />
</EBComponentsProvider>

// Resume existing client (clientId from provider)
<EBComponentsProvider clientId="0030000133" apiBaseUrl="/api" headers={{}}>
  <OnboardingFlow
    availableProducts={['EMBEDDED_PAYMENTS']}
    availableJurisdictions={['US']}
  />
</EBComponentsProvider>
```

### With Event Handlers

```tsx
<OnboardingFlow
  availableProducts={['EMBEDDED_PAYMENTS']}
  availableJurisdictions={['US']}
  onPostClientSettled={(data, error) => {
    if (data) console.log('Client created:', data.id);
  }}
  onPostPartySettled={(data, error) => {
    if (data) console.log('Party added:', data.id);
  }}
  onGetClientSettled={(data, status, error) => {
    console.log('Client fetch status:', status);
  }}
/>
```

### With User Tracking

```tsx
<OnboardingFlow
  availableProducts={['EMBEDDED_PAYMENTS']}
  availableJurisdictions={['US']}
  userEventsHandler={(context) => {
    analytics.track(context.actionName, {
      ...context.metadata,
      timestamp: context.timestamp,
    });
  }}
/>
```
